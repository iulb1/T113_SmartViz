# Qt线程教学 - 基于音频录制播放项目

## 1. Qt线程基础

### 1.1 为什么需要多线程？

在GUI应用程序中，如果所有操作都在主线程（UI线程）中执行，会导致：
- **界面冻结**：长时间操作会阻塞UI，用户无法交互
- **用户体验差**：无法显示进度，无法取消操作
- **响应慢**：UI无法及时响应用户操作

**解决方案**：将耗时操作放到工作线程中执行，主线程只负责UI更新。

### 1.2 Qt线程类

Qt提供了多种线程相关的类：
- **QThread**：最常用的线程类，用于创建新线程
- **QThreadPool**：线程池，管理多个线程
- **QtConcurrent**：高级API，简化并发编程
- **QMutex, QWaitCondition**：线程同步工具

本项目主要使用 **QThread**。

## 2. QThread使用方式

### 2.1 继承QThread方式（本项目使用的方式）

#### 步骤1：创建线程类

```cpp
// recordthread.h
#ifndef RECORDTHREAD_H
#define RECORDTHREAD_H

#include <QThread>
#include <QString>

class RecordThread : public QThread
{
    Q_OBJECT  // 必须包含，才能使用信号槽

public:
    explicit RecordThread(QObject *parent = nullptr);
    void stopRecording();  // 公共方法，用于停止线程
    
signals:
    void recordFinished(bool success, const QString &message);
    void recordProgress(int seconds);
    
protected:
    void run() override;  // 重写run()方法，这是线程的执行入口
    
private:
    bool m_stopFlag;  // 停止标志
};

#endif // RECORDTHREAD_H
```

**关键点**：
- 继承自 `QThread`
- 必须包含 `Q_OBJECT` 宏
- 重写 `run()` 方法，这是线程的执行入口
- 使用信号（signals）与主线程通信

#### 步骤2：实现线程逻辑

```cpp
// recordthread.cpp
#include "recordthread.h"

RecordThread::RecordThread(QObject *parent)
    : QThread(parent), m_stopFlag(false)
{
}

void RecordThread::stopRecording()
{
    m_stopFlag = true;  // 设置停止标志
}

void RecordThread::run()
{
    // 这个方法会在新线程中执行
    // 所有耗时操作都在这里
    
    // 1. 初始化操作
    // 2. 循环处理
    while (!m_stopFlag) {
        // 执行具体任务
        // ...
        
        // 发送进度信号
        emit recordProgress(seconds);
    }
    
    // 3. 清理资源
    // 4. 发送完成信号
    emit recordFinished(success, message);
}
```

**关键点**：
- `run()` 方法中的代码在新线程中执行
- 使用 `m_stopFlag` 控制线程退出
- 使用 `emit` 发送信号通知主线程

#### 步骤3：在主线程中使用

```cpp
// widget.cpp
void Widget::on_btnRecord_clicked()
{
    // 1. 创建线程对象
    m_recordThread = new RecordThread(this);
    
    // 2. 连接信号槽
    connect(m_recordThread, &RecordThread::recordFinished, 
            this, &Widget::onRecordFinished);
    connect(m_recordThread, &RecordThread::recordProgress,
            this, &Widget::onRecordProgress);
    connect(m_recordThread, &RecordThread::finished, 
            this, [this]() {
                if (m_recordThread) {
                    m_recordThread->deleteLater();
                    m_recordThread = nullptr;
                }
            });
    
    // 3. 启动线程
    m_recordThread->start();
}

void Widget::on_btnStopRecord_clicked()
{
    if (m_recordThread && m_recordThread->isRunning()) {
        m_recordThread->stopRecording();  // 请求停止
    }
}
```

**关键点**：
- 使用 `start()` 启动线程（会自动调用 `run()`）
- 使用信号槽进行线程间通信
- 使用 `deleteLater()` 安全删除线程对象

## 3.线程锁

### 3.1 基本概念

**互斥锁（Mutex）** 是一种同步机制，用于保护**临界资源**，确保同一时刻只有一个线程能够访问被保护的资源。

**工作原理**

~~~
线程A：尝试获取锁 → 成功 → 执行临界区代码 → 释放锁
线程B：尝试获取锁 → 等待（阻塞） → 等待... → 锁释放 → 获取锁 → 执行临界区代码 → 释放锁
~~~

### 3.2 为什么需要互斥锁

**多线程访问共享资源的问题**

当多个线程同时访问同一个资源时，可能出现：

1. **数据竞争（Race Condition）**
   - 多个线程同时读写同一数据
   - 导致数据不一致或损坏
2. **资源冲突**
   - 多个线程同时操作硬件设备
   - 导致设备状态混乱
3. **文件访问冲突**
   - 一个线程写入，另一个线程读取
   - 可能导致读取到不完整的数据

### 3.3 QT互斥锁QMutex

QMutex 是 Qt 的互斥锁，用于保护临界资源，确保同一时刻只有一个线程能访问。

创建互斥锁

~~~c
// widget.h
QMutex *m_recordMutex;  // 声明互斥锁指针

// widget.cpp
m_recordMutex = new QMutex();  // 创建互斥锁对象
~~~

传递线程锁给线程

~~~c
// widget.cpp
RecordThread::setRecordMutex(m_recordMutex);  // 将锁传递给所有录制线程

// recordthread.h
static QMutex *s_recordMutex;  // 静态成员，所有录制线程共享同一个锁
//### 4. 为什么使用静态成员变量？static QMutex *s_recordMutex; *// 静态成员*
//- 所有录制线程共享同一个锁对象
//- 确保多个线程竞争同一个资源时能正确互斥
~~~

在线程中使用锁

- 手动锁需要手动 unlock()，避免忘记释放

- 异常退出时也要释放锁（在 return 前调用 unlock()）

- 使用 tryLock() 可以立即检测锁状态，不会阻塞

~~~c
// recordthread.cpp - run()函数中

// 1. 尝试获取锁（非阻塞）
if (!s_recordMutex->tryLock()) {
    // 获取失败，说明锁被占用
    qDebug() << "无法获取锁，线程退出";
    return;  // 线程结束
}

// 2. 获取成功，执行录制操作
// ... 录制代码 ...

// 3. 释放锁（手动解锁）
s_recordMutex->unlock();
~~~

**关键方法**

| 方法      | 说明       | 特点                                              |
| :-------- | :--------- | :------------------------------------------------ |
| tryLock() | 尝试获取锁 | 非阻塞，立即返回<br>成功返回 true，失败返回 false |
| lock()    | 获取锁     | 阻塞，会等待直到获取到锁                          |
| unlock()  | 释放锁     | 手动释放，必须配对使用                            |
