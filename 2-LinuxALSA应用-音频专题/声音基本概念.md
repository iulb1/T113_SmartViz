# 声音基本概念

## 一、声音格式

### 1.1 常见音频格式

#### PCM (Pulse Code Modulation - 脉冲编码调制)
- **定义**: 数字音频的原始格式，未经压缩
- **特点**: 文件较大，音质无损，常用于专业音频处理
- **应用**: 音频处理、录音、专业制作

#### WAV (Waveform Audio File Format)
- **定义**: Windows系统标准音频格式，基于PCM
- **特点**: 支持多种编码方式，最常见的是PCM编码
- **文件结构**: 文件头 + PCM数据

#### MP3 (MPEG-1 Audio Layer 3)
- **定义**: 有损压缩音频格式
- **特点**: 文件小，音质较好，广泛使用
- **压缩比**: 通常为原始PCM的1/10左右

#### AAC (Advanced Audio Coding)
- **定义**: 高级音频编码，MP3的改进版
- **特点**: 相同码率下音质优于MP3
- **应用**: 移动设备、流媒体

#### FLAC (Free Lossless Audio Codec)
- **定义**: 无损压缩音频格式
- **特点**: 压缩率约50%，音质无损
- **应用**: 音乐收藏、专业音频

#### OGG
- **定义**: 开源音频容器格式
- **特点**: 支持多种编码，常用于游戏音频

### 1.2 格式分类

**按压缩方式分类:**

- **无损格式**: PCM, WAV, FLAC, APE
- **有损格式**: MP3, AAC, OGG, WMA

**按应用场景分类:**

- **专业制作**: PCM, WAV, FLAC
- **日常使用**: MP3, AAC
- **流媒体**: AAC, OGG

~~~shell
# 什么是FFmpeg
# FFmpeg 是跨平台的多媒体处理工具，用于录制、转换和流式传输音视频。

# 更新软件包列表
sudo apt update
# 安装ffmpeg
sudo apt install ffmpeg
# 验证安装
ffmpeg -version

#录制PCM
ffmpeg -f alsa -ar 44100 -ac 2 -i default -t 10 -f s16le source.pcm
# 转换为WAV
ffmpeg -f s16le -ar 44100 -ac 2 -i source.pcm source.wav
# 转换为FLAC
ffmpeg -f s16le -ar 44100 -ac 2 -i source.pcm -c:a flac source.flac

# 转换为MP3（不同码率）
ffmpeg -f s16le -ar 44100 -ac 2 -i source.pcm -b:a 128k source_128k.mp3
# 转换为AAC
ffmpeg -f s16le -ar 44100 -ac 2 -i source.pcm -c:a aac -b:a 128k source.aac
# 转换为AAC
ffmpeg -f s16le -ar 44100 -ac 2 -i source.pcm -c:a libvorbis -q:a 5 source.ogg


#播放pcm
ffmpeg -f s16le  -ar 44100 -ac 2 -i source.pcm  -f alsa default


~~~

## 二、音频参数

### 2.1 采样率 (Sample Rate)

**定义**: 每秒采集声音样本的次数，单位：Hz (赫兹)

- 单位：Hz（赫兹）

- 含义：每秒对音频信号进行采样的次数

- 8000 Hz = 每秒采集 8000 个样本点
- 每秒钟对声音信号采样 8000 次

- 每两个采样点之间的时间间隔 = 1/8000 秒 = 0.000125 秒 = 125 微秒

- 这是电话质量的采样率，适合语音通话

**常见采样率:**

- **8000 Hz**: 电话质量
- **11025 Hz**: 低质量音频
- **22050 Hz**: 广播质量
- **44100 Hz**: CD质量（最常用）
- **48000 Hz**: 专业音频、视频
- **96000 Hz**: 高保真音频
- **192000 Hz**: 超高保真音频

**采样定理**: **采样率必须至少是信号最高频率的2倍（奈奎斯特定理）**

需要注意区分两个概念：

采样率（Sample Rate）：每秒采集样本的次数

- 例如：8000 Hz = 每秒8000次采样

音频频率（Audio Frequency）：声音本身的频率

- 例如：440 Hz = 标准A音的音调

奈奎斯特定理：采样率必须至少是信号最高频率的 2 倍，才能完整还原信号。

- 8000 Hz 采样率 → 最高可还原 4000 Hz 的音频频率

- 44100 Hz 采样率 → 最高可还原 22050 Hz 的音频频率（人耳可听范围约 20-20000 Hz）



### 2.2 位深度 (Bit Depth / 采样精度)

**定义**: 每个采样点用多少位来表示，决定动态范围

**常见位深度:**
- **8位**: 256个级别，电话质量
- **16位**: 65536个级别，CD质量（最常用）
- **24位**: 16777216个级别，专业音频
- **32位**: 浮点数，专业音频处理

**动态范围计算**: 动态范围 ≈ 6.02 × 位深度 (dB)

- 8位: 约48 dB
- 16位: 约96 dB
- 24位: 约144 dB

### 2.3 声道数 (Channels)

**定义**: 音频通道的数量

**常见声道配置:**
- **单声道 (Mono)**: 1个声道
- **立体声 (Stereo)**: 2个声道（左、右）
- **5.1声道**: 6个声道（前置左、前置右、中置、低音、环绕左、环绕右）
- **7.1声道**: 8个声道

### 2.4 码率 (Bitrate)

**定义**: 每秒传输的比特数，单位：bps (bits per second)

**常见码率:**

- **MP3**: 128 kbps（标准质量）、192 kbps（高质量）、320 kbps（最高质量）
- **AAC**: 128 kbps（相当于MP3的192 kbps）
- **PCM**: 无压缩，码率 = 采样率 × 位深度 × 声道数

### 2.5 音频参数总结表

| 参数 | 单位 | 常见值 | 说明 |
|------|------|--------|------|
| 采样率 | Hz | 44100, 48000 | 每秒采样次数 |
| 位深度 | bit | 16, 24, 32 | 每个采样的精度，多少bit来表示一个采样点 |
| 声道数 | 个 | 1, 2, 5.1, 7.1 | 音频通道数量 |
| 码率 | bps | 128k, 192k, 320k | 每秒数据量（压缩格式） |

## 三、音频文件大小计算

### 3.1 PCM/WAV文件大小计算

**基本公式:**
```
文件大小(字节) = 采样率 × 位深度/8 × 声道数 × 时长(秒)
```

**示例计算:**

**例1: CD质量音频（44.1kHz, 16bit, 立体声）**

- 采样率: 44100 Hz
- 位深度: 16 bit = 2 字节
- 声道数: 2
- 时长: 60 秒

```
文件大小 = 44100 × 2 × 2 × 60
         = 10,584,000 字节
         ≈ 10.09 MB
```

**例2: 专业音频（48kHz, 24bit, 立体声）**

- 采样率: 48000 Hz
- 位深度: 24 bit = 3 字节
- 声道数: 2
- 时长: 60 秒

```
文件大小 = 48000 × 3 × 2 × 60
         = 17,280,000 字节
         ≈ 16.48 MB
```

**例3: 单声道电话质量（8kHz, 8bit, 单声道）**
- 采样率: 8000 Hz
- 位深度: 8 bit = 1 字节
- 声道数: 1
- 时长: 60 秒

```
文件大小 = 8000 × 1 × 1 × 60
         = 480,000 字节
         ≈ 0.46 MB
```

### 3.2 压缩格式文件大小计算

**基本公式:**
```
文件大小(字节) = 码率(bps) × 时长(秒) / 8
```

**示例计算:**

**MP3 128kbps, 时长3分钟:**

```
文件大小 = 128000 × 180 / 8
         = 2,880,000 字节
         ≈ 2.75 MB
```

**AAC 192kbps, 时长5分钟:**

```
文件大小 = 192000 × 300 / 8
         = 7,200,000 字节
         ≈ 6.87 MB
```

### 3.3 数据率计算

**PCM数据率:**

```
数据率(bps) = 采样率 × 位深度 × 声道数
```

**示例:**
- 44.1kHz, 16bit, 立体声: 44100 × 16 × 2 = 1,411,200 bps ≈ 1.41 Mbps
- 48kHz, 24bit, 立体声: 48000 × 24 × 2 = 2,304,000 bps = 2.304 Mbps

### 3.4 常用音频文件大小参考

| 格式 | 参数 | 1分钟大小 | 1小时大小 |
|------|------|-----------|-----------|
| PCM | 44.1kHz/16bit/立体声 | ~10 MB | ~600 MB |
| PCM | 48kHz/24bit/立体声 | ~16.5 MB | ~990 MB |
| WAV | 44.1kHz/16bit/立体声 | ~10 MB | ~600 MB |
| MP3 | 128 kbps | ~0.94 MB | ~56 MB |
| MP3 | 192 kbps | ~1.41 MB | ~85 MB |
| MP3 | 320 kbps | ~2.35 MB | ~141 MB |
| AAC | 128 kbps | ~0.94 MB | ~56 MB |
| FLAC | 44.1kHz/16bit/立体声 | ~5 MB | ~300 MB |

## 四、音频处理相关概念

### 4.1 音频缓冲区

**定义**: 临时存储音频数据的内存区域

**大小计算:**

```
缓冲区大小(字节) = 采样率 × 位深度/8 × 声道数 × 时长(秒)
```

**示例: 1秒缓冲区（44.1kHz/16bit/立体声）**

```
缓冲区大小 = 44100 × 2 × 2 × 1
           = 176,400 字节
           ≈ 172 KB
```

### 4.2 音频延迟

**计算延迟:**

```
延迟(秒) = 缓冲区大小(字节) / (采样率 × 位深度/8 × 声道数)
```

### 4.3 音频帧

**定义**: 一次处理的最小音频数据单元

**帧大小计算:**

```
帧大小(字节) = 采样数 × 位深度/8 × 声道数
```

**示例: 1024个采样（44.1kHz/16bit/立体声）**

```
帧大小 = 1024 × 2 × 2
       = 4,096 字节
       = 4 KB
```

## 五、总结

1. **采样率**决定时间分辨率，越高越精细
2. **位深度**决定动态范围，越高细节越丰富
3. **声道数**决定空间感，立体声提供左右定位
4. **码率**（压缩格式）决定压缩后的质量
5. **文件大小** = 采样率 × 位深度/8 × 声道数 × 时长（PCM）
6. **文件大小** = 码率 × 时长 / 8（压缩格式）

选择合适的参数需要在**音质**、**文件大小**和**处理能力**之间平衡。
